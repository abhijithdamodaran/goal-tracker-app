# Project Specification — Goal Tracker App

Source: Ouroboros interview `interview_20260419_173920` → seed `seed_2b1b4da50866` (ambiguity 0.177).

## Product summary

A web-first goal tracking app with AI coaching. Enables individuals and couples
to create SMART goals, decompose them into milestones and action items across
sprint and cycle cadences, track recurring habits with configurable streaks,
and share goals with a spouse through a co-editable family workspace.

## Users and workspaces

- Every user always has a **personal workspace**.
- A user may additionally belong to **at most one family group** (spouses only — no kids, no child accounts, no COPPA handling).
- Family membership is **symmetric co-edit**: any family member can edit the goal, its milestones, action items, and habits.
- Families are created by one user. Members are added via **invite link / single-use code** (8-character code, 7-day expiry, single-use) generated by the creator.

## Domain hierarchy

```
Goal ─▶ Milestone ─▶ ActionItem
         (Cycle)      (weekly Sprint)

Goal ─▶ Habit  (parallel track, recurring, streak-tracked)
               (optionally attachable to a Sprint or Cycle for short-term habits)
```

- **Goal** — SMART goal with title, description, SMART score (0–5), ownership (personal or family), time-bound deadline.
- **Milestone** — intermediate target under a Goal, assigned to a **Cycle** of monthly / quarterly / half-yearly / yearly length.
- **ActionItem** — one-off task under a Milestone with done / not-done state, assigned to a **weekly Sprint**.
- **Habit** — recurring action with a schedule, streak mode, current streak count. Long-term by default. Short-term habits MAY be attached to a specific Sprint or Cycle.
- **Sprint** — weekly time-boxed iteration (Mon–Sun). Contains assigned ActionItems. Ends with a **sprint review screen**.
- **Cycle** — longer time window (monthly, quarterly, half-yearly, yearly) scoping Milestones.

## SMART validation

- **Advisory only — never blocks save.**
- Per-element inline hints for each SMART dimension (Specific, Measurable, Achievable, Relevant, Time-bound).
- Overall SMART **score 0–5** with improvement suggestions.
- Soft warnings for missing fields only.

## Habit streaks

Each habit is configured with **one of three streak modes** (user-selectable per habit):

1. **Strict** *(default)* — any missed scheduled occurrence resets streak to 0.
2. **One-grace-per-week** — one skip per week is tolerated without breaking the streak. A second miss in the same week resets.
3. **Percentage-threshold** — streak survives while weekly completion rate stays at or above a configured threshold (e.g. 80%).

Streak weeks run **Monday–Sunday** (ISO week). The `getWeekStart()` utility in `src/lib/streaks/utils.ts` implements this via `daysBack = (dow + 6) % 7`.

## Authentication

- **Google SSO** (requires `GOOGLE_CLIENT_ID` + `GOOGLE_CLIENT_SECRET`)
- **Apple SSO** (requires `APPLE_CLIENT_ID` + `APPLE_CLIENT_SECRET`)
- **Magic link** (passwordless email via Nodemailer; falls back to console output in dev)
- **Password login** (Credentials provider with bcrypt hashing — for dev convenience)

All four offered on the `/signin` page. Session strategy: **JWT** (required for Credentials provider; Auth.js v5 Credentials does not support database sessions).

Dev test user: `john@example.com` / `password123`. Recreate with `node scripts/seed-dev-user.mjs`.

## Platform

- **MVP: web-first**, family-adoptable polish (clean UI, onboarding flow, error states — usable by a non-technical spouse without coaching).
- **v2: native mobile** (iOS + Android) with push notifications.
- Push notifications are explicitly **out of v1**.

## Tech stack

| Language | TypeScript |

## Integrations

- **Google Calendar** and **Apple Calendar** sync for action item deadlines and sprint events. (Direction: at minimum push-to-calendar; bidirectional if feasible.)
- **Offline support** required: daily check-ins (habit ticks, action item completions) must work offline and **background-sync** when connectivity returns. Conflict resolution: last-write-wins on `recordedAt` timestamp.

## AI coaching

- On **new goal creation**, the app suggests a breakdown plan: recommended milestones, action items, and a timeline.
- The app is positioned as a **coach**, not a passive tracker — ongoing guidance toward goal attainment is expected.
- Model: `claude-sonnet-4-6` (fast, cost-effective). Use prompt caching.

## Home screen

**Configurable per user**, default = Today view. Three available views (same regardless of role):

1. **Today** — today's habits with streak state + today's action items from current sprint.
2. **Sprint Board** — current weekly sprint: all action items + habits grouped by day.
3. **Goal Dashboard** — all active goals with milestone progress, current sprint status, recent habit streaks.

## Acceptance criteria (MVP smoke tests)

1. A user creates a family, invites their spouse via invite link, creates a SMART goal with milestones, and sees a SMART score **≥ 4 / 5**.
2. A user creates a weekly sprint, assigns 3 action items from a milestone, completes them during the week, and sees a sprint review screen.
3. A user sets a habit to **one-grace-per-week** mode, misses one scheduled day and the streak is preserved, misses a second day in the same week and the streak resets to 0.
4. A user creates a new goal and receives an AI-suggested breakdown with milestones, action items, and timeline.
5. Action item deadlines and sprint events sync to Google Calendar or Apple Calendar.
6. A user completes a habit check-in while offline and the data syncs correctly when connectivity returns.
7. A new user signs up via any auth method, creates or joins a family via invite link/code, and reaches the Today view without external help.
8. The home screen can be switched between Today, Sprint Board, and Goal Dashboard.
9. Milestones can be assigned to monthly, quarterly, half-yearly, or yearly cycles.
10. Habits track streaks in the user-selected mode (strict / one-grace-per-week / percentage-threshold).

- Push notifications.
- Native mobile apps.
- Child accounts, parental controls, COPPA handling.
- Multiple family groups per user.

- Hard timeline / deadline for shipping (ship when it's right).

| `Sprint` | Weekly time-box (Mon–Sun); holds ActionItems; reviewedAt for sprint review |
| `ActionItem` | Task under a Milestone; assigned to a Sprint; done/not-done toggle; ownerId + assigneeId |

**Implementation notes:**

## Phase 9 — Integrations (Calendar + Offline) ⬜

**Goal:** Action item deadlines appear in Google/Apple Calendar; habit check-ins work offline.

**Deliverables:**

### Calendar sync
- Google OAuth scope: `https://www.googleapis.com/auth/calendar.events`
- On action item create/update: push event to Google Calendar (or Apple Calendar via CalDAV)
- On sprint create: push sprint "block" event covering Mon–Sun
- Settings toggle: enable/disable calendar sync per calendar type
- Sync status indicator on action items (synced / pending / error)

### Offline support
- Service worker (Workbox) installed at app root
- Habit check-ins and action item done-toggles queued in IndexedDB when offline
- Background sync: flush queue to `/api/habits/[id]/check-ins` and `/api/action-items/[id]` when connection restored
- Conflict resolution: last-write-wins based on `recordedAt` timestamp (already in HabitCheckIn model)
- Offline banner in nav when no connection detected

**Acceptance:** AC 5 (calendar) + AC 6 (offline sync).
